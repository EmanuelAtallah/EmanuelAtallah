Features

This project is a console-based Pac-Man-inspired survival game. The player collectes keys while avoiding monsters that actively chase the player around the map. The game includes a scoring system, multiple lives, increasing difficulty, pressure plates that remove enemies, random enemy spawning, wrap-around movement, colored game elements, and a live dashboard displaying game statistics such as score, lives, monsters, and collected keys. As the player's score increases, more monsters appear and their movement speed incresases, creating a progessively more challenging experience.

The Code

This project was built using object-oriented and procedural programming concepts. Arrays were used to manage multiple monsters, keys, and pressure plates simultaneously. The game logic was divided into methods responsible for movement, collision detection, spawning systems, difficulty scaling, rendering, and game state management.

Monster AI was implemeneted using directional movement logic that constantly moves enemies toward the player's position. Timers and the StopWatch class were used to control monster movement independently from player input, creating smoother gameplay. Randomized spawining systems were created using the Random class while also preventing enemies from spawning too close to the player. I also implemented a replay system, live dashboard updates, and colored console rendering to improve the overall user experience.

How It Turned Out

Overall, this project turned out very successfully and became one of the most advanced console applications I had created at the time. It helped me improve my understanding of game loops, real-time input handling, arrays, debugging, and program organization. One of the biggest challenges was managin multiple monsters while keeping movement smooth and preventing visual glitches on the console screen. Through this project, I gained experience building larger programs with multiple interacting systems.
